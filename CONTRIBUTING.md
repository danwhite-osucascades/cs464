# Contributing

We’re keeping things pretty simple, but there are a few guidelines to follow so everything stays organized.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes

Example:
git checkout -b feature/your-feature-name

## Making Changes

- Try to keep your changes focused on one thing
- Write clear and readable code
- Add comments if something is not obvious
- Test your code before submitting

## Submitting a Pull Request

1. Push your branch to your fork
2. Open a Pull Request to the main repository
3. Give a clear description of what you changed and why

Try to:
- Explain the purpose of your changes
- Reference any related issues

## Code Style

There’s no super strict style guide, but:
- Keep formatting consistent
- Use meaningful variable names
- Follow existing patterns in the project

## Final Notes

The goal is to make the project better, not more complicated.
